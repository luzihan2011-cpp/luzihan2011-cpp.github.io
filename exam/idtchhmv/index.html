题目描述

小信的课题组最近在基因重组上获得了重大突破。我们用小写字母表示基因，他们发现，如果有一串字母通过重新排列可以得到另一串字母，那么我们称其为可突变串。

输出格式

题目描述

为了战胜敏家军，小信建议曹操将战船用铁链相连，现在我们可以将战船和铁链组成的集合看做是一张平面图（即任意两条铁链不相交），其中战船可以看做是点，铁链可以看做是边，每条战船拥有一个点权，代表战船的战力。现在小信想请你找到，最大的特种兵团战力值。我们定义一个特种兵团为一个由若干战船和铁链组成的子集，其中任意两点之间都有直接的边相连，并且内部的权值之和是最大的。

为了迎合新规，小信决定找到一个连通的子图（这里根据题意修正理解，原描述“任意两点之间都有直接的边相连”应为连通，否则是完全图，但样例可能不符 ），求这个连通子图的点权和最大值，且要保留输入给定的铁链连接关系（即子图的边是原铁链中满足连接子图内点的部分 ），并且要保证该子图构成一个连通的平面图（题目给定条件，输入的铁链构成平面图，我们找的子图也需是连通的 ）。

题目描述

题目描述

小信购买了一套不同尺寸的角色布。由于手工染色太费时间，他设计了一套巨型自动染坊，流程如下：
1. 将所有布按选定顺序排列在传送带上（顺序可以任意选定）
2. 选择颜色 CCC 和数字 FFF（FFF 必须小于目前颜色为 CCC 的布的数量）
3. 从左到右扫描：将所有颜色为 CCC 的布重新上色。前 FFF 块颜色为 CCC 的布会被染上新颜色 XXX，其余颜色 CCC 的布块染上新颜色 YYY，XXX 和 YYY 由机器自动选择，彼此不同且不与任何现有颜色重复。此步骤消耗的墨水等于被染色的布尺寸的总和
4. 重复步骤2和3，直到所有布颜色互不相同

小信的路标

注

题目描述

题目描述

作为一名优秀的木匠，小信最近收到了一笔精巧的订单，需制作符合特定样式的器材，样式规则如下：

  • 每一块新箭头的尾端必须与前一块垂直对齐，要么与箭头的基座对齐，要么与另一箭头的尾端对齐，两块模板必须有重叠部分（我们将箭头开始的地方称为箭头基座 ）。
  • 小信唯一能确定记录的设计是发送的整型编码，但这些数字串并不能确定唯一模型。这个包含 N+1N+1N+1 个元素的编码记录了螺丝从底部到顶部的 NNN 个箭头，第一个元素表示底部箭头左侧的位置，其余 NNN 个元素定义了箭头基座的位置（由下而上）；第 iii 个元素表示第 iii 个箭头基座的位置，例如图中两种器材都可以用 2 6 5 1 42 \ 6 \ 5 \ 1 \ 42 6 5 1 4 表示。

现在请你根据小信记录的数字序列，计算有多少种可能的器材。由于答案很大，请对 232−12^{32} - 1232−1 取模，序列保证第二个数字大于第一个（底部箭头固定指向右 ）。

输入格式

第一行包含整数 NNN，第二行是 N+1N+1N+1 个的整数序列，同一行的数字用单个空格分隔。

输出格式

输出一行，表示你能排列描述成的不同器材数量，对 232−12^{32} - 1232−1 取模。

样例

6 6 5 1 4 3

Output 1

样例 2

Input 2

由于题目中 1≤N≤30001 \leq N \leq 30001≤N≤3000 使 O(n2)O(n^2)O(n2) 的复杂度可以通过此题，
所有我们可以设计一个 DP 通过此题。

我们规定 DP[i][j] 表示第 iii 个箭头的尾在第 jjj 个输入处的方案数。
设计转移:

  • 当 i = j 时:
    第 iii 个箭头的尾部在第 i−1i-1i−1 个箭头的基座处。
    显然，他可以让尾端指向基座。
    但在这之前，我们需要考虑: 怎样的组合才合法呢？
    我们显然不能这样:

    由此我们发现，必须指向不同:

    // in 是输入
for (int k = 1; k <= j; k++)
{
    if ((a[i] < a[j] && a[i + 1] > a[j]) || (a[i] > a[j] && a[i + 1] < a[j]))
        DP[i][j] += DP[i - 1][k];
}

  • 当 i != j 时:
    第 iii 个箭头的尾部在第 j−1j-1j−1 个箭头的基座处。
    那就只能向「尾部也在第 j−1j-1j−1 个箭头基座处的状态」继承。
    并且我们不能这样:

    由此我们发现，必须指向相同:

    if ((a[i] < a[j] && a[i + 1] < a[j]) || (a[i] > a[j] && a[i + 1] > a[j])) // 同向
    DP[i][j] += DP[i - 1][j];

对此，我们发现复杂度来到了 O(n3)O(n^3)O(n3)
可以优化一下:

for (ll j = 1; j <= i; j++)
{
    if (j == i)
        dp[i][j] = (dp[i][j] + f[i - 1][!(a[i + 1] > a[j])]) % mod;
    if (j < i)
    {
        if ((a[i] < a[j] && a[i + 1] < a[j]) || (a[i] > a[j] && a[i + 1] > a[j])) // 同向
            dp[i][j] = (dp[i][j] + dp[i - 1][j]) % mod;
    }
}

for (ll j = 1; j <= i; j++)
{
    if (a[i + 1] < a[j])
        f[i][0] = (f[i][0] + dp[i][j]) % mod;
    else if (a[i + 1] != a[j])
        f[i][1] = (f[i][1] + dp[i][j]) % mod;
}

即，将所有同向的方案存到另一个
